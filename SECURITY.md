# Security Policy

## Reporting a vulnerability

If you find a security issue in `autoapp-toolkit` itself (a script that leaks secrets, an env-var that gets logged, etc.):

- Email **jiejuefuyou@gmail.com** with subject `[autoapp-toolkit security]`
- Or open a GitHub issue marked **draft** so it's not public initially
- I'll respond within 7 days

Do **not** open a public issue with reproducible details until a fix is available.

## What this toolkit is in scope for

- Scripts that handle ASC API keys / GitHub PATs / fastlane match passwords
- Logic that determines what gets pushed where (CI workflow definitions)
- Memory templates that the agent reads on every session

What it is **not** in scope for:
- Vulnerabilities in your specific app code (those go to your app's repo)
- Third-party dependencies (`fastlane`, `gh`, `xcodegen`, `playwright`) — report upstream
- Apple Developer / App Store Connect issues — Apple handles those

## Sensitive data handling

The toolkit's design principle: **secrets never enter this repo**, in scripts, in commits, or in CI logs.

If you find a code path where a script could log a secret to stdout or a file: that's a vulnerability. Report it.

## What you should know about your own setup

The toolkit assumes:

- Your `~/AuthKey.p8` (ASC API key) is in your home dir, mode 600
- Your GitHub PAT is in your local environment via `gh auth login` or `GH_PAT` env var
- Your `MATCH_PASSWORD` is in `~/.autoapp-match-password` (mode 600), generated by `setup-asc-secrets.sh` on first run

If any of these get into a shared environment, rotate immediately:
- ASC: `App Store Connect > Users and Access > Keys` — revoke + regenerate
- GitHub PAT: `github.com/settings/tokens` — revoke + regenerate
- MATCH_PASSWORD: regenerate locally + run `bash scripts/setup-asc-secrets.sh` to push the new password to GitHub Actions secrets across all repos

## A note on running scripts that other people wrote

The toolkit is small (16 files at v1.0). You can read every script in 30 minutes. Do that before running any script that handles your ASC API key. The scripts are MIT-licensed because they're meant to be inspected, not just consumed.
